# Vague Code AI Partner Problem: Exam Preparation Summary for Complete Beginners

## Main Goal
Learn why vague instructions to AI coding agents result in code that looks good but fails in real-world usage. Understand the importance of Specification-Driven Development - building solid foundations before creating user-facing features.

## Section 1: Core Definitions

- **Vibe Coding**: Developing software by loose, intuitive descriptions without precise requirements (like telling a chef "make me something delicious" and expecting a perfect meal)
- **Specification-Driven Development**: Building detailed requirements and constraints before writing code (like creating detailed blueprints before construction begins)
- **AI Coding Agent**: AI tool that writes code based on your instructions (like a very skilled but literal-minded apprentice who does exactly what you say)
- **Foundation vs. Interior**: Code that handles core functionality vs. surface features (Foundation is the house's structure; interior is the paint and furniture)
- **Collaborative Specification**: Working with AI to discover and document requirements together (like brainstorming with a knowledgeable partner who asks questions you didn't think of)

## Section 2: Key Concepts (Structured with Bullet Points)

- **Vague instructions lead to problematic code**: AI agents do exactly what you say, not what you mean
- **House-building analogy**: Beautiful houses with poor foundations are unusable, just like software with great UI but poor backend
- **Sandra's failure**: Beautiful app that crashed when multiple users tried to sign up simultaneously
- **AI differs from search engines**: AI coding agents need explicit requirements, unlike search engines that return "good enough" results
- **Collaborative approach works**: Working with AI to build specifications together discovers requirements you didn't think of
- **Alex's success**: Solid foundation resulted in code that worked correctly on the first try

## Section 3: Step-by-Step Processes

1. **Identify the problem**: Recognize that vague instructions lead to code that fails in real-world usage
2. **Apply the house analogy**: Focus on building solid foundations (core functionality) before interior (surface features)
3. **Avoid "vibe coding"**: Replace loose, intuitive descriptions with precise requirements
4. **Use collaborative specification**: Ask AI "What questions should I answer to make this specification complete?"
5. **Build the foundation first**: Develop core functionality before user-facing features
6. **Discover requirements together**: Use AI as a partner to uncover edge cases and missing requirements
7. **Implement specification-driven development**: Create complete specifications before writing code

## Section 4: Importance/Relevance

This lesson is crucial because 85% of AI coding projects fail due to poor requirements, not because AI can't code. When you skip specification work, you save time upfront but spend 5-10x more time fixing problems later. In the age of AI agents, the developers who succeed will be those who know how to collaborate with AI on foundations before asking for features.
